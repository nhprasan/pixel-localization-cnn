Objective: Use deep learning model to predict the (x, y) coordinate position of the single bright/white pixel (value 255) in a 5050 grayscale image where all other pixels are 0.

Nature of the Problem Space: The problem space contains only 2,500 mathematically distinct samples, since each image is uniquely defined by the position of a single white pixel in a 5050 grid, and each position corresponds to exactly one unique image. Total possible positions=5050=2,500 images. Therefore, the dataset is finite and fully enumerable. The mapping from input to output is perfectly deterministic:

  • Input: One pixel at coordinate (x, y) has value 255
  • Output: The coordinate (x, y)

There is no ambiguity or stochastic behavior (no noise or variation). Each input has exactly one valid output. This creates a one-to-one mapping between image and coordinate label. We are basically trying to locate the index of the maximum pixel value (equivalent to a spatial argmax operation over the image). Thus, the model is not learning complex patterns but rather it is approximating a deterministic spatial lookup function.

NOTE: Because the space is fully bounded and contains only 2,500 unique inputs:

  • Memorization is sufficient to solve the task.
  • Generalization is trivial, since all possible configurations can be enumerated.
  • Even a shallow model can achieve near-zero loss.
  • A deep learning model is technically over-parameterized for this problem. Deep learning is used here primarily to demonstrate supervised regression workflows rather than because of inherent problem complexity.

This makes the task fundamentally different from real-world vision problems, where data is continuous, noisy, and highly variable. The problem becomes meaningfully complex only when variations are introduced in the data (data augmentation), at which point the model must learn robust localization rather than performing a simple lookup operation.

Dataset Generation

  • Image Dimensions: 50 50 pixels
  • Pixel Values:
    • All pixels = 0
    • One randomly selected pixel = 255
  • Single bright pixel per image
  • Labels: Store the corresponding (x, y) coordinates of the bright pixel
  • Coordinate Convention:
    • x horizontal axis (column index, increases to the right)
    • y vertical axis (row index, increases downward)
    • origin at top-left (0,0)

def generate_all_possible_samples():
    """
    Create every possible 50x50 image with one bright pixel.
    Returns 2500 images (50x50 grid positions) and their coordinates.
    Coordinates are in (x, y) format where x is column, y is row.
    """
    total_samples = IMAGE_HEIGHT * IMAGE_WIDTH
    
    images = np.zeros((total_samples, IMAGE_HEIGHT, IMAGE_WIDTH), dtype=np.uint8)
    coordinates = np.zeros((total_samples, 2), dtype=np.int32)
    
    sample_idx = 0
    
    # Loop through every pixel position in the 50x50 grid
    for y in range(IMAGE_HEIGHT):
        for x in range(IMAGE_WIDTH):
            # Start with blank image
            img = np.full(IMAGE_SHAPE, BACKGROUND_VALUE, dtype=np.uint8)
            
            # Place single bright pixel at current position
            img[y, x] = TARGET_PIXEL_VALUE
            
            images[sample_idx] = img
            coordinates[sample_idx] = [x, y]
            
            sample_idx += 1
    
    return images, coordinates


def split_data(images, coordinates, val_size, test_size, random_state):
    """
    Split dataset into train/validation/test sets.
    First split: separate test set
    Second split: divide remaining into train and validation
    This ensures proper 80/10/10 distribution.
    """
    # First split: 90% train+val, 10% test
    X_temp, X_test, y_temp, y_test = train_test_split(
        images, coordinates,
        test_size=test_size,
        random_state=random_state,
        shuffle=True
    )
    
    # Second split: from remaining 90%, take 1/9 for validation (10% of total)
    val_size_adjusted = val_size / (1 - test_size)
    X_train, X_val, y_train, y_val = train_test_split(
        X_temp, y_temp,
        test_size=val_size_adjusted,
        random_state=random_state,
        shuffle=True
    )
    
    splits = {
        'train': {'images': X_train, 'coordinates': y_train},
        'val': {'images': X_val, 'coordinates': y_val},
        'test': {'images': X_test, 'coordinates': y_test}
    }
    
    return splits


def save_data_to_disk(splits, data_dir):
    """
    Save each split as compressed numpy file for later loading.
    Using .npz format keeps file sizes small.
    """
    for split_name, split_content in splits.items():
        filepath = data_dir / f"{split_name}_data.npz"
        
        np.savez_compressed(
            filepath,
            images=split_content['images'],
            coordinates=split_content['coordinates']
        )
        
        print(f"Saved {split_name} split: {filepath}")
        print(f"  Images: {split_content['images'].shape}")
        print(f"  Coordinates: {split_content['coordinates'].shape}")

Data Preprocessing

Neural networks train more efficiently when inputs are scaled to a smaller range such as: 0 to 1. Reasons for normalization:

  • Improves gradient stability
  • Prevents large weight updates
  • Accelerates convergence
  • Reduces numerical instability

Even though the data here is simple, normalization ensures consistent training dynamics and follows best practices in deep learning.

# Normalize pixel values and coordinates to [0,1] range for better training
# ============================================================================

def normalize_images(images):
    """
    Scale pixel values from [0, 255] to [0, 1].
    Neural networks perform better with smaller input values.
    """
    normalized = images.astype(np.float32) / PIXEL_MAX_VALUE
    return normalized


def normalize_coordinates(coordinates, image_shape):
    """
    Scale coordinates to [0, 1] based on image dimensions.
    For 50x50 image: coordinates range [0, 49], so divide by 49.
    This keeps target values in same range as inputs.
    """
    coords_float = coordinates.astype(np.float32)
    
    height, width = image_shape
    max_x = width - 1
    max_y = height - 1
    
    coords_float[:, 0] = coords_float[:, 0] / max_x
    coords_float[:, 1] = coords_float[:, 1] / max_y
    
    return coords_float


def denormalize_coordinates(normalized_coords, image_shape=(50, 50)):
    """
    Convert normalized coordinates back to pixel values.
    Multiply by (dimension - 1) and round to nearest integer. Used for visualization and evaluation.
    """
    coords = normalized_coords.copy()
    height, width = image_shape
    
    coords[:, 0] = coords[:, 0] * (width - 1)
    coords[:, 1] = coords[:, 1] * (height - 1)
    
    coords = np.round(coords).astype(np.int32)
    coords[:, 0] = np.clip(coords[:, 0], 0, width - 1)
    coords[:, 1] = np.clip(coords[:, 1], 0, height - 1)
    
    return coords


def load_raw_data(split_name, data_dir):
    """Load raw data from disk."""
    filepath = data_dir / f"{split_name}_data.npz"
    
    if not filepath.exists():
        raise FileNotFoundError(f"Data file not found: {filepath}")
    
    data = np.load(filepath)
    return {'images': data['images'], 'coordinates': data['coordinates']}


def preprocess_and_save(split_name, data_dir, output_dir):
    """
    Load, normalize, and save preprocessed data.
    Normalization uses image dimensions to ensure train/val/test are scaled identically.
    """
    raw_data = load_raw_data(split_name, data_dir)
    image_shape = raw_data['images'].shape[1:]
    
    normalized_images = normalize_images(raw_data['images'])
    normalized_coords = normalize_coordinates(raw_data['coordinates'], image_shape)
    
    preprocessed = {
        'images': normalized_images,
        'coordinates': normalized_coords
    }
    
    output_path = output_dir / f"{split_name}_preprocessed.npz"
    np.savez_compressed(
        output_path,
        images=preprocessed['images'],
        coordinates=preprocessed['coordinates']
    )
    
    print(f"{split_name}: {len(normalized_images)} samples - "
          f"Image range [{normalized_images.min():.3f}, {normalized_images.max():.3f}] - "
          f"Coord range [{normalized_coords.min():.3f}, {normalized_coords.max():.3f}]")
    
    return preprocessed

class PixelLocatorDataset(Dataset):
    """
    PyTorch Dataset for pixel localization task.
    
    Handles shape transformation:
    - Input images: (50, 50) -> (1, 50, 50) for CNN
    - Coordinates: (2,) remain as is
    """
    
    def __init__(self, images, coordinates):
        self.images = images
        self.coordinates = coordinates
    
    def __len__(self):
        return len(self.images)
    
    def __getitem__(self, idx):
        """
        Get single sample and convert to tensors.
        Add channel dimension for CNN: (50, 50) -> (1, 50, 50)
        """
        img = self.images[idx]
        coords = self.coordinates[idx]
        
        img_tensor = torch.from_numpy(img).float()
        coords_tensor = torch.from_numpy(coords).float()
        
        # Add channel dimension for grayscale image
        img_tensor = img_tensor.unsqueeze(0)
        
        return img_tensor, coords_tensor


def load_preprocessed_data(split_name, preprocessed_dir):
    """Load normalized data from disk."""
    filepath = preprocessed_dir / f"{split_name}_preprocessed.npz"
    
    if not filepath.exists():
        raise FileNotFoundError(f"File not found: {filepath}")
    
    data = np.load(filepath)
    return {'images': data['images'], 'coordinates': data['coordinates']}


def create_dataloaders(batch_size=32):
    """
    Create DataLoaders for all splits.
    
    DataLoader benefits:
    - Automatic batching
    - Shuffling for train set (improves generalization)
    - No shuffling for val/test (consistent evaluation)
    """
    # Load all splits
    train_data = load_preprocessed_data('train', PREPROCESSED_DIR)
    val_data = load_preprocessed_data('val', PREPROCESSED_DIR)
    test_data = load_preprocessed_data('test', PREPROCESSED_DIR)
    
    # Create datasets
    train_dataset = PixelLocatorDataset(
        train_data['images'],
        train_data['coordinates']
    )
    val_dataset = PixelLocatorDataset(
        val_data['images'],
        val_data['coordinates']
    )
    test_dataset = PixelLocatorDataset(
        test_data['images'],
        test_data['coordinates']
    )
    
    # Create DataLoaders
    train_loader = DataLoader(
        train_dataset,
        batch_size=batch_size,
        shuffle=True,
        num_workers=0
    )
    val_loader = DataLoader(
        val_dataset,
        batch_size=batch_size,
        shuffle=False,
        num_workers=0
    )
    test_loader = DataLoader(
        test_dataset,
        batch_size=batch_size,
        shuffle=False,
        num_workers=0
    )
    
    return {
        'train_loader': train_loader,
        'val_loader': val_loader,
        'test_loader': test_loader,
        'train_dataset': train_dataset,  # Return datasets too
        'val_dataset': val_dataset,
        'test_dataset': test_dataset
    }

Model Architecture

CNNs are particularly well-suited for image data because:

  • They preserve spatial structure
  • They use parameter sharing, reducing total parameters
  • They are translation-aware (feature detection independent of position) In this problem, the location of the white pixel is spatial information. A CNN can learn filters that activate when a high-intensity pixel appears in a region of the image, making it a natural architectural choice for image-based regression.

The architecture is intentionally kept shallow because:

  • The task is simple (only one active pixel).
  • The input resolution is small (5050).
  • The mapping from input to output is deterministic and low complexity.

Therefore, a small number of convolutional layers followed by fully connected layers is sufficient to:

  • Extract spatial features
  • Map those features to continuous coordinate outputs (x, y)

The final layer uses a linear activation because this is a regression problem predicting continuous values.

# ============================================================================
# CNN MODEL ARCHITECTURE
# Build convolutional neural network for pixel localization
# ============================================================================

class PixelLocatorCNN(nn.Module):
    """
    CNN architecture for predicting pixel coordinates.
    
    Design rationale:
    - 3 conv layers progressively extract spatial features
    - Each conv layer uses 3x3 kernels (standard for feature extraction)
    - MaxPool reduces spatial dimensions while keeping important features
    - Final layers compress to 2 outputs: x and y coordinates
    
    Architecture flow:
    Input (1, 50, 50) -> Conv+Pool -> (16, 25, 25) -> 
    Conv+Pool -> (32, 12, 12) -> Conv+Pool -> (64, 6, 6) -> 
    Flatten -> FC(128) -> Output(2)
    """
    
    def __init__(self):
        super(PixelLocatorCNN, self).__init__()
        
        # First conv block: extract basic features
        # padding=1 keeps spatial dimensions same before pooling
        self.conv1 = nn.Conv2d(
            in_channels=1,
            out_channels=16,
            kernel_size=3,
            padding=1
        )
        self.pool1 = nn.MaxPool2d(kernel_size=2, stride=2)
        
        # Second conv block: combine basic features into patterns
        self.conv2 = nn.Conv2d(
            in_channels=16,
            out_channels=32,
            kernel_size=3,
            padding=1
        )
        self.pool2 = nn.MaxPool2d(kernel_size=2, stride=2)
        
        # Third conv block: high-level feature extraction
        self.conv3 = nn.Conv2d(
            in_channels=32,
            out_channels=64,
            kernel_size=3,
            padding=1
        )
        self.pool3 = nn.MaxPool2d(kernel_size=2, stride=2)
        
        # Flattened size after all pooling: 64 channels * 6 * 6
        self.flatten_size = 64 * 6 * 6
        
        # Dense layer to learn coordinate relationships
        self.fc1 = nn.Linear(self.flatten_size, 128)
        
        # Output layer: 2 neurons for (x, y) with no activation
        # Linear activation is appropriate for regression
        self.fc_output = nn.Linear(128, 2)
        
        # ReLU adds non-linearity for better feature learning
        self.relu = nn.ReLU()
    
    
    def forward(self, x):
        """
        Forward pass through network.
        Each conv block: convolution -> activation -> pooling
        Then flatten and pass through dense layers.
        """
        x = x.to(torch.float32)
        
        # Conv block 1
        x = self.conv1(x)
        x = self.relu(x)
        x = self.pool1(x)
        
        # Conv block 2
        x = self.conv2(x)
        x = self.relu(x)
        x = self.pool2(x)
        
        # Conv block 3
        x = self.conv3(x)
        x = self.relu(x)
        x = self.pool3(x)
        
        # Flatten for dense layers
        x = x.reshape(-1, self.flatten_size)
        
        # Dense layer with activation
        x = self.fc1(x)
        x = self.relu(x)
        
        # Output layer (no activation for regression)
        x = self.fc_output(x)
        
        return x


def create_cnn_model():
    """
    Initialize model with loss function and optimizer.
    
    Loss: MSE is standard for regression tasks
    Optimizer: Adam adapts learning rate automatically
    """
    model = PixelLocatorCNN()
    criterion = nn.MSELoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=LEARNING_RATE)
    
    return model, optimizer, criterion

Note on Baseline Models and Experimental Practice

In a typical machine learning workflow, it is good practice to begin with a simple baseline model before introducing more specialized architectures.

For this type of problem, a Multi-Layer Perceptron (MLP) could serve as a baseline because:

  • The 5050 image can be flattened into a 2500-dimensional input vector.
  • The relationship between the single active pixel and its coordinates is straightforward.
  • No complex spatial hierarchy is required for solving the task.

Including an MLP baseline would allow us to:

  • Compare its performance against a CNN.
  • Follow a structured experimentation pipeline (baseline specialized model comparison).

# Define training and validation loops

def train_epoch(model, train_loader, criterion, optimizer, device):
    """
    Train model for one epoch.
    
    Steps per batch:
    1. Forward pass: compute predictions
    2. Calculate loss
    3. Backward pass: compute gradients
    4. Update weights
    """
    model.train()
    total_loss = 0.0
    num_batches = 0

    # Progress bar for batches
    pbar = tqdm(train_loader, desc="Training", leave=False)
    
    for images, coordinates in pbar:
        images = images.to(device)
        coordinates = coordinates.to(device)
        
        # Reset gradients from previous batch
        optimizer.zero_grad()
        
        # Forward pass
        predictions = model(images)
        loss = criterion(predictions, coordinates)
        
        # Backward pass and weight update
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item()
        num_batches += 1

        # Update progress bar with current batch loss
        pbar.set_postfix({'batch_loss': f'{loss.item():.6f}'})
    
    return total_loss / num_batches


def validate_epoch(model, val_loader, criterion, device):
    """
    Evaluate model on validation set.
    No gradient computation needed for evaluation.
    """
    model.eval()
    total_loss = 0.0
    num_batches = 0

    # Progress bar for validation batches
    pbar = tqdm(val_loader, desc="Validating", leave=False)
    
    with torch.no_grad():
        for images, coordinates in pbar:
            images = images.to(device)
            coordinates = coordinates.to(device)
            
            predictions = model(images)
            loss = criterion(predictions, coordinates)
            
            total_loss += loss.item()
            num_batches += 1

            pbar.set_postfix({'batch_loss': f'{loss.item():.6f}'})
    
    return total_loss / num_batches


def train_model(model, train_loader, val_loader, criterion, optimizer, num_epochs, device):
    """
    Complete training loop with validation.
    """
    history = {
        'train_loss': [],
        'val_loss': [],
        'epoch_times': []
    }
    
    total_start_time = time.time()
        
    # Main epoch progress bar
    epoch_pbar = tqdm(range(num_epochs), desc="Overall Progress")
    
    for epoch in epoch_pbar:
        epoch_start = time.time()
        
        # Train and validate
        train_loss = train_epoch(model, train_loader, criterion, optimizer, device)
        val_loss = validate_epoch(model, val_loader, criterion, device)
        
        epoch_time = time.time() - epoch_start
        
        # Record history
        history['train_loss'].append(train_loss)
        history['val_loss'].append(val_loss)
        history['epoch_times'].append(epoch_time)
        
        # Update main progress bar
        epoch_pbar.set_postfix({
            'train_loss': f'{train_loss:.6f}',
            'val_loss': f'{val_loss:.6f}'
        })
        
        # Print epoch summary
        print(f"Epoch {epoch+1}/{num_epochs} - "
              f"Train: {train_loss:.6f} - "
              f"Val: {val_loss:.6f} - "
              f"Time: {epoch_time:.2f}s")
    
    total_training_time = time.time() - total_start_time
    
    # Print comprehensive training summary
    print()
    print("="*70)
    print("TRAINING COMPLETED - SUMMARY")
    print("="*70)
    
    # Basic metrics
    print(f"\n{'Metric':<30} {'Value':>15}")
    print("-"*70)
    print(f"{'Total Training Time':<30} {total_training_time:>12.2f}s")
    print(f"{'Average Epoch Time':<30} {np.mean(history['epoch_times']):>12.2f}s")
    print(f"{'Total Epochs':<30} {num_epochs:>15d}")
    
    # Loss metrics
    print(f"\n{'Loss Metrics':<30} {'Train':>15} {'Validation':>15}")
    print("-"*70)
    print(f"{'Initial Loss':<30} {history['train_loss'][0]:>15.6f} {history['val_loss'][0]:>15.6f}")
    print(f"{'Final Loss':<30} {history['train_loss'][-1]:>15.6f} {history['val_loss'][-1]:>15.6f}")
    
    return history

# Test Set Evaluation
# Evaluate final model performance on unseen test data

def test_model(model, test_loader, criterion, device):
    """
    Evaluate model on test set and collect predictions for analysis.
    Returns average loss and all predictions with ground truth.
    """
    model.eval()
    total_loss = 0.0
    num_batches = 0
    
    all_predictions = []
    all_ground_truth = []
    all_images = []
    
    with torch.no_grad():
        for images, coordinates in test_loader:
            images = images.to(device)
            coordinates = coordinates.to(device)
            
            predictions = model(images)
            loss = criterion(predictions, coordinates)
            
            total_loss += loss.item()
            num_batches += 1
            
            # Store predictions for later analysis
            all_predictions.append(predictions.cpu().numpy())
            all_ground_truth.append(coordinates.numpy())
            all_images.append(images.cpu().numpy())
    
    # Combine batches into single arrays
    predictions = np.concatenate(all_predictions, axis=0)
    ground_truth = np.concatenate(all_ground_truth, axis=0)
    test_images = np.concatenate(all_images, axis=0)
    
    avg_loss = total_loss / num_batches
    
    return avg_loss, predictions, ground_truth, test_images
